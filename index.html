【BI工具】Redash的部署和使用系列（二）：安装部署

2019-07-12 #redash #Python
我这边是用官方提供的docker方式部署的，也有直接部署在系统环境中的，这里推荐一篇文章：Redash开发指南，他有个微信群，大家可以在文章里加下，如果有对二次开发有需求的同学； 官方给了Ubuntu18.04的一键安装脚本：奈何国内网络环境的问题，基本上会很慢很慢很慢，所以为了方便，我读了shell文件后，决定自己安装，当然还是在docker中部署； （使用的Ubuntu，其他系统基本一致，大家主要看逻辑，如果遇到一些问题可以自己google，基本都不会是大问题，也可以加tg群：点我加群，我们一起研究解决） 步骤一： 安装docker和docker-compose，docker可以直接在官网下载安装，或者apt或者yum怎么方便怎么来 Mac版本和Windows版本已经集成了docker-compose，linux好像没有； docker-compose如果显示命令找不到，说明就没有，google下安装就好 这里注意下，要设置下docker的加速代理，不然pull文件时会很慢； 我用的阿里云的代理，速度杠杠的 这是地址阿里云容器镜像服务，打开登录后,左下角有个镜 像加速器，就能看到自己的加速链接了，一人一码，不要钱 不仅如此，页面中把加速器的配置方法都写上了，我在这里就不阐述了 步骤二： 1、建立好配置文件夹： sudo mkdir /opt/redash 2、这里很关键！！ 先建立一个env 文件： sudo vim /opt/redash/env 这里记录的是一些数据库连接url和密码之内的信息，很好理解： PYTHONUNBUFFERED=0 REDASH_LOG_LEVEL=INFO REDASH_REDIS_URL=redis://redis:6379/0 POSTGRES_PASSWORD=数据库密码（redash自己的数据库哦) REDASH_COOKIE_SECRET=[REDASH_COOKIE_SECRET](自己写) REDASH_SECRET_KEY=[_SECRET_key] REDASH_DATABASE_URL=postgresql://postgres:[数据库密码]@postgres/postgres 其实上面的信息并不是很重要，可以用脚本生成随机的参数 REDASH_BASE_PATH=/opt/redash COOKIE_SECRET=$(pwgen -1s 32) SECRET_KEY=$(pwgen -1s 32) POSTGRES_PASSWORD=$(pwgen -1s 32) REDASH_DATABASE_URL=&quot;postgresql://postgres:${POSTGRES_PASSWORD}@postgres/postgres&quot; echo &quot;PYTHONUNBUFFERED=0&quot; &gt;&gt; $REDASH_BASE_PATH/env echo &quot;REDASH_LOG_LEVEL=INFO&quot; &gt;&gt; $REDASH_BASE_PATH/env echo &quot;REDASH_REDIS_URL=redis://redis:6379/0&quot; &gt;&gt; $REDASH_BASE_PATH/env echo &quot;POSTGRES_PASSWORD=$POSTGRES_PASSWORD&quot; &gt;&gt; $REDASH_BASE_PATH/env echo &quot;REDASH_COOKIE_SECRET=$COOKIE_SECRET&quot; &gt;&gt; $REDASH_BASE_PATH/env echo &quot;REDASH_SECRET_KEY=$SECRET_KEY&quot; &gt;&gt; $REDASH_BASE_PATH/env echo &quot;REDASH_DATABASE_URL=$REDASH_DATABASE_URL&quot; &gt;&gt; $REDASH_BASE_PATH/env 新建个sh 文件 粘贴上面的代码运行就好了 会生成一个env 好了，到第三步： docker安装主要用的是docker-compose.yml文件来做启动的，这个文件非常重要，他设置了所需要的启动的所有容器和基本配置 这里我做了些修改： 修改内容： 1、指定了redash版本 2、增加了redis和postgres的db文件与宿主机的映射，不让docker容器停止后数据丢失 3、增加了redis和postgres的端口映射，方便调试，线上环境也可以关掉 4、修改REDASH_COOKIE_SECRET 5、增加了邮箱设置 6、增加了对python Query 的支持 废话不多说：我直接贴docker-compose.yml上来 version: '2' x-redash-service: &amp;redash-service image: redash/redash:7.0.0.b18042 depends_on: - postgres - redis env_file: /opt/redash/env restart: always services: server: &lt;&lt;: *redash-service command: server ports: - &quot;5000:5000&quot; depends_on: - server links: - server:redash restart: always 新建个yml文件，把以上代码根据自己的信息修改好之后填进去，存在/opt/redash下面，命名为：docker-compose.yml 以上准备工作都已经完成！！！ 最后一步： 执行两个条命令： sudo docker-compose run --rm server create_db sudo docker-compose up -d 第一条是创建新的数据库 第二条是启动 会跑一会，等全部执行完毕，就好了 直接输入你部署的ip到浏览器就可以到管理员的设置页面了 ！！！
